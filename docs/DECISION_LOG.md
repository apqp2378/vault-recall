# 결정 로그 — vault-recall

> knowledge-ops의 전통 계승: 모든 방향 전환은 근거와 함께 기록한다.

## D-001 · 두 레포를 하나로 흡수한다
- topic-shelf(수집·큐레이션)와 knowledge-ops(그래프 진단)는 같은 문제("저장한 지식이 소환되지 않는다")의 앞뒤 반쪽이었다.
- 수집(ingest) → 구조화(parse/graph) → 소환(recall) → 진단(diagnose) → 증빙(eval)의 한 루프로 통합.

## D-002 · 엣지는 '실제 위키링크'다 (knowledge-ops의 최대 약점 교정)
- 구버전 엣지는 'GitHub topic 공유'라는 근사치 → 관계의 의미가 약했다.
- Obsidian 볼트의 [[위키링크]]는 사람이 승인한 진짜 관계. 이것을 1급 엣지로 쓴다.
- 결과: 그래프 부스트 검색(연결된 노트는 함께 소환)이 가능해짐.

## D-003 · 코어는 100% 결정적, LLM은 확장점
- 검색(BM25)·그래프·진단·평가 전부 표준 라이브러리, 같은 입력=같은 출력.
- 임베딩(BGE-M3)과 LLM 요약은 provider 확장점(topic-shelf 패턴 계승)으로만.
- 이유: 신뢰(재현 가능한 숫자) + 어디서나 실행(의존성 0) + '생성은 선택, 판정은 사람'.

## D-004 · 정직한 공백(honest gap)을 1급 기능으로
- 근거 점수가 임계 미만이면 '근거 부족'을 선언하고, 볼트가 커버 못한 질의 어절을 그대로 보여준다.
- 할루시네이션을 모델 튜닝이 아니라 구조(소환만 하고 생성하지 않음)로 차단. (ragflow/khoj의 인용 철학)

## D-005 · 품질 주장은 골드셋 숫자로 증빙
- "검색이 잘 된다"는 주장 대신 gold.json 기반 Recall@k·MRR을 CLI로 재현 가능하게. (ragas 개념의 결정적 축소판)
- 골드셋에 정답이 볼트에 없는 질의(간격반복 등)를 일부러 포함 — 공백 선언이 작동하는지 확인.

## D-006 · 개인 데이터는 레포에 넣지 않는다
- 테스트·데모는 합성 demo_vault로. 개인 볼트(career-vault) 실측치는 숫자만 README에 인용.


## D-007 · 기능별 효과크기는 절제 실험으로 확정한다 (2차 검토)
- 골드셋 20건(볼트 내 정답) 절제 실험: 음절 bigram이 최대 효과(R@5 +10%p, MRR +0.092),
  MOC 패널티 0.5가 +0.017, **그래프 부스트는 랭킹 효과 0**.
- 그래프 부스트는 제거하지 않는다 — 랭킹이 아니라 '근거 확장'(연결 카드 동반 소환)이 존재 이유임을 문서화.
- 교훈: 기능의 존재 이유와 측정 지표를 일치시켜라. 랭킹 지표로 정당화되지 않는 기능은 다른 지표(맥락 제공)로 설명하거나 빼라.

## D-008 · 공백 판정은 스케일 불변 비율로 (절대 임계 폐기)
- 절대 점수 임계(2.0)는 분리 실패: in-vault 최저 top점수 17.7 vs out-vault 최고 17.4 — 사실상 미작동.
- top점수/이상점수(질의 전 토큰 tf=1 상한) 비율은 분리 성공: in 중앙값 1.23 vs out 중앙값 0.17.
- 3단계(충분 ≥0.6 / 약함 ≥0.3 / 공백 <0.3)로 경계 사례를 '약함 경고'로 정직하게 처리.
- 평가 기준도 갱신: 정답 없는 질의는 '비확신(약함·공백)'이면 성공, '확신 있는 오답'만 실패.

## D-009 · 임베딩은 auto 기본화 + 우아한 폴백 (로드맵 1 완료)
- 기본 모델 multilingual-e5-small(한국어·경량), e5 프리픽스 자동, .recall_cache 디스크 캐시.
- 미설치·다운로드 차단·OOM 등 어떤 실패에도 코어(BM25+그래프)로 폴백 — 코어는 완결적.
- 이 개발 환경은 모델 다운로드가 차단돼 융합 효과의 실측은 로컬 몫 —
  대신 갭을 측정해 동봉: 패러프레이즈 10건에서 코어 R@5 60% (eval/gold_paraphrase.json).


## D-010 · 참고 10선 전수 반영 감사 (누락 4건 교정)
- 감사 결과 10선 중 4건이 미반영/절반 반영이었다: docling(문서 수집)·리랭커·ts-fsrs(훈련)·근거 하이라이트.
- 반영: ① `ingest --files` — md/txt 내장 파서 + pdf/docx는 docling optional provider(미설치 시 안내 후 스킵)
  ② `--rerank` — cross-encoder 재채점(bge-reranker-v2-m3 기본, 우아한 폴백)
  ③ `train` — SM-2 간격반복(결정적, 채점은 사람 grade 0~5, .recall_cache/srs_log.json). FSRS는 provider 교체 경로.
  ④ `report --query` — 소환 근거를 지식지도에 하이라이트("근거 카드를 불빛으로").
  ⑤ lint 값 검증(type 열거·verified 불리언·priority 형식) — instructor 개념 심화.
- 원칙 유지: 새 기능 전부 결정적이거나(train·lint·highlight) 우아한 폴백을 가진 optional provider(docling·reranker).
- 훈련 카드 선정 우선순위: 기한 지난 카드 → 미훈련 answer/experience 먼저 (면접 대비 가치 기준, 결정적).
